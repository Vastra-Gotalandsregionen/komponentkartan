<vgr-header> är en komponent som skall användas i huvudet på sidorna och ligga där fast när användaren navigerar i systemet.
Färgen kan sättas beroende på vilket system som är aktivt. I portalen beror färgen på om användaren har ett eller flera system. Om bara ett system finns, är det den färgen som gäller. Headern innehåller en transparent bild som gör att man bara behöver sätta headerns bakgrundsfärg genom rätt klassnamn på headern eller på ett föräldraelement. I det "neutrala" läget skall inte den transparenta bilden synas. Headern kan visas i olika utföranden, man kan ha den med eller utan användarinformation samt med eller utan meny och olika kombinationer av dessa. I exemplet kan man se de olika vyerna.

På headern kan man även lägga till en kontextmeny ( <vgr-header-menu>). Den menyn kan endast bo i headern.

Properties vgr-header

Namn Beskrivning Exempel
headerMenu : IHeaderMenu Menyn man vill visa (IHeaderMenu) [headerMenu]="contextMenu"
userName : string Sätter namnet på texten för den som är inloggad [userName]="signedInUser"

IHeaderMenu

Namn Beskrivning Exempel
menuItems: IHeaderMenuItem[] menyalternativ.

IHeaderMenuItem

Namn Beskrivning Exempel
displayName: string Menyalternativets rubrik displayName: 'Min sida'
url: string Länk att följa vid klick på länken. url: "/my-route"
isInternalLink: boolean Anger om alternativet skall öppnas i ett nytt fönster eller ej. isInternalLink: true
isSeparator: boolean Anger om alternativet är en horisontell linje eller ej. isSeparator: true
menuItems: IHeaderMenuItem[] Eventuella underalternativ för menyalternativet. OBS! endast en undernivå är möjlig
