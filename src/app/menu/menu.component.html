Komponentkartan använder en vänstermeny, som skapas med <vgr-sidebar-menu>. Vänstermenyn består av en eller flera menyer, som i sin tur innehåller menyalternativ med eller utan underalternativ. Det är även möjligt att lägga ut avdelare mellan menyalternativ för att skapa sammanhang.

Singel-meny

En applikation med ett system har en meny som täcker hela ytan vertikalt.

Multi-meny

En applikation med flera system innehåller flera menyer som kan öppnas och stängas.

Avdelare

För att indikera att menyalternativ är orelaterade kan en avdelare läggas in

Underalternativ

Ett menyalternativ kan ha underalternativ som kan visas och döljas

Tema

Varje meny har ett tema, baserat på det aktuella systemets färg. För applikationer med flera system, använd property theme på <vgr-menu>. För applikationer med ett system styrs tema av det övergripande temat som sätts "längst ut" i applikationen. Se Temahantering

Markup

<vgr-sidebar-menu> har en property som behöver sättas, [menus]. Menyer är en lista av IMenu, och är tänkt att hämtas från servern, som då kan bestämma vilka menyalternativ en användare får i systemet. När [menus] sätts behandlas listan internt av komponenten som sätter ytterligare properties på den. Nedan listas de properties som är aktuella att fylla i för användande system.


IMenu

Namn Beskrivning Exempel
title: string menyns rubrik title: 'System'
groups: IMenuGroup[] Grupperade menyalternativ, separeras med vertikal linje

IMenuGroup

Namn Beskrivning Exempel
order: string Soteringsordning för gruppen. Grupper visas efter order i alfanumerisk sortering order: '1'
menuItems: IMenuItem[] Gruppens menyalternativ

IMenuItem

Namn Beskrivning Exempel
title: string Menyalternativets rubrik title: 'Min sida'
url: string Länk att följa vid klick på länken. url: "/my-route"
order: string Menyalternativets ordning, alternativets. Sorteras alfanumeriskt. order: "2"
menuItems: IMenuItem[] Eventuella underalternativ för menyalternativet. OBS! endast en undernivå är möjlig
